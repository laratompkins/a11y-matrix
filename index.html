Accessibility Matrix for College of DuPage Brand Color Palette

When we use our brand palette, we must ensure that the colors we choose for text and graphics have enough contrast to the background color. Doing so means that the materials we create will be accessible to the largest audience. The table below lists all brand color combinations. It shows which pairings have the required level of contrast for different uses. These uses include:

  • graphics and user interface elements
  • text and images of text.

View all examples of color combinations suitable for: Graphical objects, text minimum (AA and AAA), and text enhanced (AAA).

Explaining the matrix.
Rating Explanation
F FAIL - 224 of 324 combinations (69%).
  • Color combination should not be used for user interface (UI) components or graphical objects where no text is used, nor for the visual representation of text.
G Contrast ratio is at least 3:1 - 100 of 324 combinations (31%).
  • Suitable for User Interface Components and Graphical Objects where no text is used (WCAG 2.1 Success Criterion 1.4.11 Non-text Contrast).
  • Suitable for large-scale text (at least 18 point if not bold and at least 14 point if bold) and images of large-scale text (WCAG 2.1 Success Criterion 1.4.3 Contrast (Minimum).
  • View all examples of brand color combinations with a ratio of 3:1 and above.
AA Contrast ratio is at least 4.5:1 - 62 of 324 combinations (19%).
  • Suitable for the visual representation of text and images of text to minimum level (WCAG 2.1 Success Criterion 1.4.3 Contrast (Minimum).
  • Suitable for large-scale text (at least 18 point if not bold and at least 14 point if bold) and images of large-scale text (WCAG 2.1 Success Criterion 1.4.6 Contrast (Enhanced)
  • Suitable for User Interface Components and Graphical Objects where no text is used (WCAG 2.1 Success Criterion 1.4.11 Non-text Contrast)
  • View all examples of brand color combinations with a ratio of 4.5:1 and above.
AAA Contrast ratio is at least 7:1 - 34 of 324 combinations (10%).
  • Suitable for the visual representation of text and images of text at enhanced level (WCAG 2.1 Success Criterion 1.4.6 Contrast (Enhanced).
  • Suitable for User Interface Components and Graphical Objects where no text is used (WCAG 2.1 Success Criterion 1.4.11 Non-text Contrast).
  • View all examples of brand color combinations with a ratio of 7:1 and above.
NOTE Exclusions for visual representation of text are:
  • Incidental: Text or images of text that are part of an inactive user interface component, that are pure decoration, that are not visible to anyone, or that are part of a picture that contains significant other visual content, have no contrast requirement.
  • Logotypes: Text that is part of a logo or brand name has no contrast requirement.

Credits

When you use colors from a defined palette, such as a corporate brand, you should make accessible choices. This color accessibility matrix helps you do this. A custom bash script created by Matthew Deeprose, Senior Learning Designer at the University of Southampton, generated this web page.


Additional tools used are:

  • WebAIM Color Contrast API.
  • JQ by Stephen Dolan and others.
  • csvtool by Jan Loeffler.
  • Tabulate.sh by Nathan S.R.
  • DaltonLens SVG filters.
  • System UIcons by Corey Ginnivan.
  • jQuery to help make some of the interactions easier to implement.

Whilst I have attempted to follow good practice and make this content as accessible as possible, there is always room for improvement. Send any feedback about the matrix and any suggestions for improvements to Matthew Deeprose by connecting on LinkedIn.
